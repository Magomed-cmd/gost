# Заметки для v4 промпта и доработки либы

## Формулы

| ✅ Как надо | ❌ Как не надо |
|---|---|
| `formulaMath()` для standalone выражений | `body("α = max_i min_j q_ij")` |
| `paragraphWithMath()` для смешанного текста | `body("min(M1)=3 ≥ max(M2)=3")` |
| `mathFraction`, `mathSub`, `mathSup` и т.д. для всех символов | Юникод-символы ≥ ≤ → ≈ голым текстом внутри body() |

Правило: любое выражение с переменными, индексами, дробями или
математическими символами — только через math API.

---

## Таблицы

| ✅ Как надо | ❌ Как не надо |
|---|---|
| `mathFraction()` прямо в ячейку — библиотека обрабатывает | Строка "1/2" вместо `mathFraction("1","2")` |
| Ширины в DXA, сумма = 9355 | `WidthType.PERCENTAGE` |
| `fontSize: 20` для больших таблиц (БР 30 строк) | Дефолтный fontSize — вылезет за поля |
| `tableCaption("Таблица N – Название")` — **над** таблицей, выравнивание ВЛЕВО | `caption()` для таблиц — он по центру, это для рисунков |

ГОСТ 7.32: подпись к таблице — **над** таблицей, **левый край**, без отступа.  
Подпись к рисунку — **под** рисунком, **по центру**.

```js
// Правильно для таблиц:
tableCaption("Таблица 1 – Платёжная матрица"),
makeTable(rows, opts),

// Правильно для рисунков:
imageBlock(...),
caption("Рисунок 1 – графический метод"),

// Через счётчик (table: true → автоматически tableCaption):
const tables  = createCaptionCounter("Таблица", { table: true });
const figures = createCaptionCounter("Рисунок");  // по умолчанию centered
```

## Текст разделов

| ✅ Как надо | ❌ Как не надо |
|---|---|
| 2–4 абзаца на подраздел: что делается, по какой формуле, что означает результат | Один `body()` на весь подраздел |
| Вывод по результату в конце подраздела | Спойлер финала в середине ("эта альтернатива получит лучший ранг") |
| Нейтральный академический текст | Технические комментарии в `body()` ("исходный код включён из файла...", "заглушка оставлена для...") |

## Имена функций

Всегда проверять реальный `module.exports` перед написанием промпта.

| В промпте было | Реально в либе |
|---|---|
| `body()` | `paragraph()` |
| `empty()` | `blank()` |
| `code()` | `codeLine()` |
| `makeTable(rows, colWidths, opts)` | `makeTable(rows, opts)` — ширины через `opts.widths` |

h1 / h2 / h3 / codeBlock / caption / placeholder / pageBreak /
imageBlock / makeTitlePage / makeDocument / makeContentSection /
saveDocument / TableOfContents — совпадают.

## Codex habits

- Читает либу независимо и проверяет реальный API — хорошо, поощрять
- Пишет meta-комментарии в `body()` — явно запретить в промпте фразой:
  "Никаких технических комментариев в body() — только академический текст"
- Пишет спойлеры финала в середине документа — запретить отдельно
- При расхождении промпта и либы — доверять либе, не промпту

## Что доработать в либе

- `imageBlock` теперь принимает и путь, и Buffer — ок, оставить
- Рассмотреть алиасы `body` / `empty` / `code` как обёртки для
  обратной совместимости с промптом v3

### createCaptionCounter — нумерация рисунков и таблиц

Добавить в либу фабрику счётчиков. Глобальный счётчик внутри модуля — нельзя
(side effects при импорте, один процесс может собирать несколько документов).

```js
function createCaptionCounter(prefix, opts = {}) {
  let n = opts.start ?? 1;
  return {
    peek:    () => n,
    caption: (text) => caption(`${prefix} ${n++} – ${text}`),
  };
}
```

Использование:

```js
const figures = createCaptionCounter("Рисунок");
const tables  = createCaptionCounter("Таблица");

body(`На рисунке ${figures.peek()} представлен графический метод.`),
imageBlock(fs.readFileSync(graphPath), 600, 420),
figures.caption("графический метод для матрицы 3×2"),
// → "Рисунок 1 – графический метод для матрицы 3×2"
```

- `peek()` — посмотреть следующий номер без увеличения
- `caption(text)` — вставить подпись и увеличить счётчик
- Не делать `current` как свойство — двусмысленно ("следующий" или "последний использованный"?)
