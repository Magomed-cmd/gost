# Future Improvements

## Numbered captions

В отчётах сейчас легко ошибиться с ручной нумерацией рисунков и таблиц: при
вставке нового элемента нужно не забыть сдвинуть все последующие подписи. Это
стоит вынести в маленький API библиотеки.

Важно: не делать module-global счётчик внутри `docx-gost.js`. Библиотека должна
оставаться без side effects при импорте, а один Node-процесс может собирать
несколько документов подряд. Правильнее создавать счётчик на один документ.

Предлагаемый API:

```js
const figures = createCaptionCounter("Рисунок");
const tables = createCaptionCounter("Таблица");

children.push(
  imageBlock(fs.readFileSync(graphPath), 600, 420),
  figures.caption("графический метод для матрицы 3×2"),
  tables.caption("матрица выигрышей"),
);
```

Ожидаемый результат:

```text
Рисунок 1 – графический метод для матрицы 3×2
Рисунок 2 – критерий Гурвица
Рисунок 3 – критерий Ходжа–Лемана
Таблица 1 – матрица выигрышей
```

Возможное расширение:

```js
const formulas = createCaptionCounter("Формула");
```

Правила реализации:

- счётчик создаётся явно через фабрику;
- начальное значение можно передать опцией;
- состояние счётчика живёт только в объекте, созданном для конкретного отчёта;
- импорт `docx-gost.js` не меняет состояние процесса;
- формат подписи должен совпадать с текущим `caption()`: `Рисунок N – текст`,
  `Таблица N – текст`.
